它是如何工作的?

cordova APP，是由cordova 命令行打包HTML、CSS、JavaScript和所有的图片，使之成为二进制文件（即.ipa或.apk文件）的一部分。一旦APP发布，再次修改代码（修改bug，或者添加新的功能）或者更新图像资源，需要重新编译和发布这个二进制文件，当然还会花费发布到应用商店的审核时间。

CodePush 插件通过保持你发布到codepush服务器上的代码和图片资源的同步更新，帮助你的改善的产品立即呈现在终端用户前。通过这种方式，你的应用程序不仅可以得到离线移动体验的好处，而且拥有一有更新就会加载的网络式体验。这是双赢。

为了保证你的终端用户的APP总有可以运行的版本，Codepush插件会维护一份以前的版本更新，以便当你提交一个不可用的更新后，它能自动回滚。通过这种方式，可以放心，在服务器上做版本回滚之前，新的更新不会对用户造成阻碍。这是三赢。

注意：涉及原始代码的改动（例如：cordova版本升级，添加新的插件），不能通过codepush插件更新，因此，这类情况需要更新应用商店。

发布更新

一旦，你完成设置的app被你的用户安装，并且你已经修改了代码和资源，这时候，你就可以立即发布它！最简单的（也是我们推荐的）方法是在CodePush命令行界面，使用 release-cordova 命令，将的更新处理并发布到Codepush服务器。

这个命令的最基本的形式只有两个参数：你的程序的名字和执行此次更新的平台（IOS或者Android）

注意：当CodePush发布更新时，你不需要更新app的config.xml中的版本号，因为你根本没有修改应用商店里的app的版本。只有当你升级Cordova或者plugin时，修改应用程序的版本，然后重新在应用商店里发布。为了帮助你标注发布历史，CodePush会生成一个标签，来标记每次发布的版本（例如：V3）。

由于知道Cordova程序的标准布局结构， release-cordova 命令可以构成一个简单的工作流程，生成更新，并且确切的知道那些文件需要上传。此外，为了灵活的支持个更新策略，release-cordova 命令提供许多可选的参数。可以定制怎样把更新展现给终端用户。（例如：哪个版本兼容此次更新？此次更新时强制性的吗？）

CodePush的客户端支持查分更新，即使你每次更新逗发布你的程序代码，你的终端用户实际上只会下载它们需要的文件。这个服务是自动的，你只需要专注于开发应程序，我们会给你分担终端用户的下载优化。

关于 release-cordova 是如果工作的细节，以及他提供的各种参数，请参考CLI 文档。另外，如果你更愿意执行 cordova prepare 命令，因而，想要一个比 release-cordova 更好的解决方案，可以参考 release 命令，获得更多细节。

参考api

在 deviceready 事件触发后，通过全局的 CodePush 对象暴露在你的程序中的CodePush API可以被使用，以下为公开的API顶层的方法：

  • checkForUpdate: 检查CodePush服务器，这个已配置的APP部署程序，是否有可用的更新。

  • getCurrentPackage: 检索当前安装的更新的元数据（例如：描述、安装时间、大小）。

  • getPendingPackage: 检索已经下载并安装，但还没有通过重启进行应用的更新的元数据。（如果有的话）

  • notifyApplicationReady: Notifies the CodePush runtime that an installed update is considered successful. If you are manually checking for and installing updates (i.e. not using the sync method to handle it all for you), then this method MUST be called; otherwise CodePush will treat the update as failed and rollback to the previous version when the app next restarts.

  • restartApplication: Immediately restarts the app. If there is an update pending, it will be immediately displayed to the end user.

  • sync: Allows checking for an update, downloading it and installing it, all with a single call. Unless you need custom UI and/or behavior, we recommend most developers to use this method when integrating CodePush into their apps.
